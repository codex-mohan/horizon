# Migration Plan: LangGraph → pi-mono

> **Scope:** Replace LangGraph/LangChain with `@mariozechner/pi-ai` + `@mariozechner/pi-agent-core` across the entire agent server and frontend chat layer.
> **Out of scope:** Chat branching, checkpoint branching — these are LangGraph-specific features we will drop per user request.

---

## Architecture Overview

### Before (LangGraph)
```
Frontend (useStream/@langchain/langgraph-sdk)
  → POST /threads/:id/runs/stream (SSE)
    → StateGraph.compile() → 6 nodes → interrupt() for HITL
      → FileSystemCheckpointer
```

### After (pi-mono)
```
Frontend (custom SSE hook)
  → POST /threads/:id/runs/stream (SSE)
    → Agent class → beforeToolCall for HITL
      → Agent message history (JSON file)
```

```typescript
/**
 * Creates a pi-ai Model object from Horizon config.
 * 
 * Key difference from LangChain:
 * - LangChain: new ChatOpenAI({ modelName, temperature, apiKey, ... })
 * - pi-ai: getModel(provider, modelName) returns metadata only
 * - Temperature, maxTokens, apiKey, reasoning are passed at stream/complete CALL TIME
 * - Custom baseUrl is set on the Model object for Ollama/NVIDIA NIM
 */
export function createModel(config: RuntimeModelConfig): Model<any> {
  const piProvider = providerMap[config.provider] || "openai";
  
  // Build custom model for providers needing baseUrl override
  let baseUrl: string | undefined;
  if (config.provider === "ollama") {
    baseUrl = config.baseUrl || "http://localhost:11434/v1";
  } else if (config.provider === "nvidia_nim") {
    baseUrl = config.baseUrl || "https://integrate.api.nvidia.com/v1";
  } else if (config.baseUrl) {
    baseUrl = config.baseUrl;
  }

  const model = getModel(piProvider as any, config.modelName);

  // For custom endpoints, create a model with baseUrl override
  if (baseUrl) {
    return {
      ...model,
      baseUrl,
      // Ollama and NVIDIA NIM are openai-completions compatible but may need compat flags
      compat: config.provider === "ollama" ? {
        supportsDeveloperRole: false,
        supportsReasoningEffort: false,
      } : undefined,
    };
  }

  return model;
}

/**
 * Build stream/complete options from RuntimeModelConfig.
 * 
 * In pi-ai, runtime options are passed at CALL TIME, not at model creation.
 * This function converts Horizon's config into pi-ai's stream/complete options format.
 */
export function buildStreamOptions(config: RuntimeModelConfig) {
  const options: Record<string, any> = {};

  if (config.apiKey) {
    options.apiKey = config.apiKey;
  }

  if (config.temperature !== undefined) {
    options.temperature = config.temperature;
  }

  if (config.maxTokens !== undefined) {
    options.maxTokens = config.maxTokens;
  }

  // Reasoning/thinking - provider-specific
  if (config.enableReasoning) {
    const provider = config.provider;
    
    if (provider === "anthropic") {
      options.thinkingEnabled = true;
      if (config.thinkingBudget) {
        options.thinkingBudgetTokens = config.thinkingBudget;
      }
    } else if (provider === "openai" || provider === "nvidia_nim") {
      if (config.reasoningEffort) {
        options.reasoningEffort = config.reasoningEffort;
      }
    } else if (provider === "google") {
      options.thinking = {
        enabled: true,
        budgetTokens: config.thinkingBudget || -1,
      };
    }
  }

  return options;
}

/**
 * Map Horizon's reasoning effort to pi-ai thinking level for Agent class.
 */
export function getThinkingLevel(
  config: RuntimeModelConfig
): "off" | "minimal" | "low" | "medium" | "high" | "xhigh" {
  if (!config.enableReasoning) return "off";
  const effort = config.reasoningEffort || "medium";
  if (effort === "minimal") return "minimal";
  if (effort === "low") return "low";
  if (effort === "high") return "high";
  if (effort === "xhigh") return "xhigh";
  return "medium";
}
```

**Key architectural differences:**

| Aspect | LangChain | pi-ai |
|---|---|---|
| Model creation | `new ChatOpenAI({...})` with all options | `getModel(provider, modelName)` returns metadata |
| API keys | Passed to constructor | Env vars auto-detected OR `apiKey` in call options |
| Temperature | Constructor option | Call-time option (`stream(model, ctx, { temperature })`) |
| Max tokens | Constructor option | Call-time option (`stream(model, ctx, { maxTokens })`) |
| Reasoning | Provider-specific constructor options | Call-time options OR `thinkingLevel` on Agent |
| Custom baseUrl | Constructor `configuration.baseURL` | Model object `baseUrl` field |
| Model object | Heavy LLM instance with methods | Lightweight metadata object |

**Implications for the migration:**

1. **AgentNode** — no longer creates LLM with `llm.bindTools(tools).invoke(messages)`. Instead, the `Agent` class handles tool binding internally.
2. **Worker** — no longer creates LLM instances. Instead, creates `Agent` instances with `initialState.model`.
3. **stream/complete calls** — all runtime options (temperature, maxTokens, apiKey, reasoning) are passed via the options parameter, not baked into the model.
4. **API key handling** — pi-ai auto-detects from env vars (`OPENAI_API_KEY`, `ANTHROPIC_API_KEY`, etc.), but for client-driven config we pass `apiKey` in call options.

### Step 3.2: Create new agent core `apps/agent/src/agent/horizon-agent.ts`

**CRITICAL: How model loading works with pi-ai's Agent class.**

Unlike LangGraph where each node creates its own LLM instance, pi-ai's `Agent` class owns the model in its state. The model is set once in `initialState.model` and can be changed by assigning `agent.state.model`. All runtime options (temperature, maxTokens, apiKey, reasoning) are passed through the Agent's `getApiKey` callback and `thinkingLevel` property — NOT through the model object itself.

```typescript
import { Agent } from "@mariozechner/pi-agent-core";
import { type Model, type Context, complete, stream } from "@mariozechner/pi-ai";
import { SYSTEM_PROMPT } from "./prompt.js";
import { tools, type AgentTool } from "./tools/index.js";
import { createModel, buildStreamOptions, getThinkingLevel } from "../lib/model.js";
import type { RuntimeModelConfig } from "../lib/model.js";
import type { ToolApprovalConfig } from "../lib/approval.js";
import { detectPII } from "../lib/pii.js";
import { ThreadStore } from "../lib/persistence.js";

export interface HorizonAgentOptions {
  threadId: string;
  modelConfig: RuntimeModelConfig;
  toolApproval: ToolApprovalConfig;
  systemPrompt?: string;
  userId?: string;
  onEvent: (event: any) => void;
}

export class HorizonAgent {
  private agent: Agent;
  private threadId: string;
  private modelConfig: RuntimeModelConfig;
  private toolApproval: ToolApprovalConfig;
  private userId?: string;
  private onEvent: (event: any) => void;
  private pendingApproval: {
    toolCallId: string;
    toolName: string;
    args: Record<string, unknown>;
    resolve: (approved: boolean) => void;
  } | null = null;

  constructor(options: HorizonAgentOptions) {
    this.threadId = options.threadId;
    this.modelConfig = options.modelConfig;
    this.toolApproval = options.toolApproval;
    this.userId = options.userId;
    this.onEvent = options.onEvent;

    // Create model from config — lightweight metadata object
    const model = createModel(this.modelConfig);

    // Load persisted messages if they exist
    const persistedMessages = ThreadStore.load(this.threadId) || [];

    this.agent = new Agent({
      initialState: {
        systemPrompt: options.systemPrompt || SYSTEM_PROMPT,
        model,
        tools: tools as AgentTool<any>[],
        messages: persistedMessages,
      },
      thinkingLevel: getThinkingLevel(this.modelConfig),
      toolExecution: "parallel",
      // Dynamic API key resolution — called per-provider before each LLM call
      getApiKey: async (provider) => {
        const config = this.modelConfig;
        if (config.apiKey) return config.apiKey;
        return undefined; // pi-ai falls back to env vars
      },
      beforeToolCall: async ({ toolCall, args, context }) => {
        // PII Detection (replaces StartMiddleware)
        if (agentConfig.ENABLE_PII_DETECTION) {
          const piiTypes = detectPII(JSON.stringify(args));
          if (piiTypes.length > 0) {
            this.onEvent({ type: "pii_detected", types: piiTypes });
          }
        }

        // HITL: Check if this tool needs approval
        if (needsApproval(toolCall.name, this.toolApproval)) {
          return new Promise<{ block: boolean; reason: string }>((resolve) => {
            this.pendingApproval = {
              toolCallId: toolCall.id,
              toolName: toolCall.name,
              args,
              resolve,
            };
            // Emit interrupt event to frontend
            this.onEvent({
              type: "interrupt",
              data: {
                action_requests: [{
                  name: toolCall.name,
                  arguments: args,
                  description: `Tool: ${toolCall.name}\nRisk: ${getToolRiskLevel(toolCall.name)}\nArgs: ${JSON.stringify(args)}`,
                }],
                review_configs: [{
                  action_name: toolCall.name,
                  allowed_decisions: ["approve", "reject"],
                }],
              },
            });
          }).then((approved) => {
            this.pendingApproval = null;
            if (!approved) {
              return { block: true, reason: `Tool ${toolCall.name} was rejected by user` };
            }
            return undefined; // approved, proceed
          });
        }

        return undefined; // auto-approved
      },
      afterToolCall: async ({ toolCall, result, isError, context }) => {
        // Emit tool execution events for generative UI
        this.onEvent({
          type: "tool_execution_end",
          data: {
            toolCallId: toolCall.id,
            toolName: toolCall.name,
            result: result.content?.[0]?.text || "",
            isError,
          },
        });
      },
    });

    // Subscribe to agent events for SSE streaming
    this.agent.subscribe(async (event) => {
      this.onEvent({ type: event.type, data: event });
    });
  }

  async prompt(content: string | any, images?: any[]) {
    // Update model if config changed since last turn
    const currentModel = createModel(this.modelConfig);
    if (this.agent.state.model.id !== currentModel.id) {
      this.agent.state.model = currentModel;
    }
    this.agent.thinkingLevel = getThinkingLevel(this.modelConfig);

    // Memory retrieval (replaces MemoryRetrieval node)
    if (this.userId && agentConfig.ENABLE_MEMORY) {
      try {
        const { initializeMemory } = await import("../agent/nodes/memory-retrieval.js");
        await initializeMemory();
        // ... retrieve and inject into system prompt
      } catch {
        // Memory not configured, skip
      }
    }

    // Build message content blocks
    const contentBlocks: any[] = [{ type: "text", text: content }];
    if (images?.length) {
      for (const img of images) {
        contentBlocks.push({ type: "image", data: img.data, mimeType: img.mimeType });
      }
    }

    return this.agent.prompt(contentBlocks.length === 1 ? content : contentBlocks);
  }

  updateModelConfig(config: RuntimeModelConfig) {
    this.modelConfig = config;
    this.agent.state.model = createModel(config);
    this.agent.thinkingLevel = getThinkingLevel(config);
  }

  updateToolApproval(config: ToolApprovalConfig) {
    this.toolApproval = config;
  }

  approvePendingToolCall() {
    if (this.pendingApproval) {
      this.pendingApproval.resolve(true);
    }
  }

  rejectPendingToolCall() {
    if (this.pendingApproval) {
      this.pendingApproval.resolve(false);
    }
  }

  get state() { return this.agent.state; }
  get isStreaming() { return this.agent.state.isStreaming; }

  abort() { this.agent.abort(); }
  async waitForIdle() { await this.agent.waitForIdle(); }

  async saveState() {
    ThreadStore.save(this.threadId, this.agent.state.messages);
  }
}
```

### Step 3.3: Create agent manager `apps/agent/src/agent/agent-manager.ts`

Manages per-thread agent instances, persistence, and SSE event streaming:

```typescript
import { HorizonAgent } from "./horizon-agent.js";
import type { RuntimeModelConfig } from "../lib/model.js";
import type { ToolApprovalConfig } from "../lib/approval.js";

class AgentManager {
  private agents = new Map<string, HorizonAgent>();

  getOrCreate(options: {
    threadId: string;
    modelConfig: RuntimeModelConfig;
    toolApproval: ToolApprovalConfig;
    userId?: string;
    onEvent: (event: any) => void;
  }): HorizonAgent {
    const existing = this.agents.get(options.threadId);
    if (existing) {
      existing.updateModelConfig(options.modelConfig);
      existing.updateToolApproval(options.toolApproval);
      return existing;
    }

    const agent = new HorizonAgent(options);
    this.agents.set(options.threadId, agent);
    return agent;
  }

  get(threadId: string): HorizonAgent | undefined {
    return this.agents.get(threadId);
  }

  async destroy(threadId: string) {
    const agent = this.agents.get(threadId);
    if (agent) {
      await agent.saveState();
      agent.abort();
      this.agents.delete(threadId);
    }
  }
}

export const agentManager = new AgentManager();
```

### Step 3.4: Handle HITL (Human-in-the-Loop)

pi-mono's `beforeToolCall` hook returns a Promise that can block execution. The approach:

1. `beforeToolCall` detects a tool needing approval
2. Creates a Promise that resolves when user responds
3. Emits `interrupt` SSE event to frontend
4. Frontend calls `/threads/:id/runs/resume` with approval decision
5. Resume handler calls `agent.approvePendingToolCall()` or `agent.rejectPendingToolCall()`
6. The Promise resolves, `beforeToolCall` returns `{ block: true }` for rejects or `undefined` for approves
7. Agent continues execution

**Important:** The `beforeToolCall` hook runs after args are validated but before tool execution. If it returns `{ block: true, reason }`, the agent reports a tool error to the LLM and continues the loop.

- `POST /threads/:threadId/runs/stream` — SSE from Agent.subscribe()
- `POST /threads/:threadId/runs/resume` — approve/reject + continue
- `POST /threads/:threadId/runs` — non-streaming invoke

### Step 4.2: New SSE streaming protocol

Instead of LangGraph's complex multi-mode events (`values`, `updates`, `messages`, `custom`), use pi-mono's native event types directly:

```
event: message_update
data: { type: "text_delta", delta: "...", messageId: "..." }

event: message_update
data: { type: "text_end", content: "...", messageId: "..." }

event: toolcall_start
data: { toolCallId: "...", toolName: "shell_execute", args: {...} }

event: tool_execution_start
data: { toolCallId: "...", toolName: "shell_execute", args: {...} }

event: tool_execution_update
data: { toolCallId: "...", status: "executing", progress: "..." }

event: tool_execution_end
data: { toolCallId: "...", toolName: "shell_execute", result: "...", isError: false }

event: interrupt
data: { action_requests: [...], review_configs: [...] }

event: agent_end
data: { messageId: "...", usage: { input: 100, output: 50, cost: 0.001 } }

event: error
data: { message: "...", type: "..." }
```

### Step 4.3: Simplified thread state format

The old `/threads/:threadId/state` endpoint returned LangGraph Platform format with checkpoints, parent configs, and nested tasks. The new format is much simpler:

```json
{
  "messages": [
    { "role": "user", "content": "Hello" },
    { "role": "assistant", "content": [{ "type": "text", "text": "Hi!" }] }
  ],
  "next": [],
  "tasks": [],
  "isStreaming": false,
  "pendingToolCalls": []
}
```

No more `checkpoint.checkpoint_id`, `parent_checkpoint`, `created_at`, or LangGraph-specific metadata.

### Step 4.4: Simplified resume endpoint

Old resume (LangGraph):
```typescript
const toolMessage = new ToolMessage(JSON.stringify({...}), APPROVAL_TOOL_NAME, APPROVAL_TOOL_NAME);
await graph.updateState({ configurable: { thread_id } }, { messages: [toolMessage] });
const resumeCommand = new Command({ resume: true });

```

New resume (pi-mono):
```typescript
if (approval?.approved) {
  agent.approvePendingToolCall();
} else {
  agent.rejectPendingToolCall();
}
await agent.continue();
// Stream events via SSE
```

## Phase 8: Subagent System

### Step 8.1: Rewrite subagent spawning

Replace the LangChain-based worker system with pi-mono's `Agent` class. **Key difference:** Workers create `Agent` instances with their own model, not raw LLM instances:

```typescript
import { Agent } from "@mariozechner/pi-agent-core";
import { createModel } from "../lib/model.js";
import { getToolsForNames } from "./tools/index.js";

async function spawnWorkers(configs: SubAgentConfig[]): Promise<SubAgentResult[]> {
  const workers = configs.map(async (config) => {
    const agent = new Agent({
      initialState: {
        systemPrompt: config.systemPrompt,
        model: createModel(config.modelConfig || fallbackModelConfig),
        tools: getToolsForNames(config.tools),
        messages: [],
      },
    });

    const startTime = Date.now();
    try {
      await agent.prompt(config.context?.task_description || config.name);
      await agent.waitForIdle();
      
      const lastMessage = agent.state.messages.at(-1);
      const output = typeof lastMessage?.content === "string" 
        ? lastMessage.content 
        : lastMessage?.content?.[0]?.text || "";

      return {
        task_id: config.id,
        agent_id: config.id,
        status: "success",
        output,
        metrics: { execution_time_ms: Date.now() - startTime },
      };
    } catch (error) {
      return {
        task_id: config.id,
        agent_id: config.id,
        status: "failure",
        output: "",
        errors: [error instanceof Error ? error.message : String(error)],
        metrics: { execution_time_ms: Date.now() - startTime },
      };
    }
  });

  return Promise.all(workers);
}
```

## Tool Approval Flow (New)

```
1. Agent calls tool → beforeToolCall hook fires
2. Hook checks needsApproval(toolName, config)
3. If needs approval:
   a. Return { block: true, reason: "..." }
   b. Agent pauses, emits interrupt SSE event
   c. Frontend shows approval dialog
   d. User approves → update pending approvals, call agent.continue()
   e. User rejects → inject toolResult with rejection message, call agent.continue()
4. If auto-approved: return undefined (proceeds normally)
```
